# 手写识别 TFLite 模型

## 模型说明

本目录用于存放中文手写字符识别的 TFLite 模型文件。

### 文件说明

- `handwriting_recognition.tflite` - TFLite 模型文件（待训练/替换）

### 模型规格

- **输入**: 64x64 灰度图像 (shape: [1, 64, 64, 1])
- **输出**: 3500 类中文字符概率分布 (shape: [1, 3500])
- **量化**: 可选 INT8 或 FLOAT32
- **标签文件**: `assets/labels.txt` (3500 个常用汉字)

### 模型架构建议

推荐使用以下架构之一：

1. **MobileNetV2** - 轻量级，适合移动端
2. **EfficientNet-Lite** - 平衡精度和速度
3. **自定义 CNN** - 针对手写字符优化

### 训练数据建议

- **CASIA-HWDB** - 中科院手写汉字数据库
- **SCUT-COUCH** - 华南理工手写数据库
- **自定义数据集** - 基于用户手写数据

### 模型文件状态

当前状态：**占位符**（非真实模型）

当真正的模型文件被替换后，TFLite 识别服务会自动检测并启用。

### 注意事项

1. 模型文件大小应 < 10MB（移动端优化）
2. 确保输入归一化方式与训练时一致（前景高值，背景低值）
3. 标签文件的行号对应输出索引（从 0 开始）
4. 模型应覆盖 labels.txt 中的所有 3500 个字符
