# Release Notes

## v4.7.0 (2026-06-23)

### 🔍 模糊图片去模糊 — 拍抖了也能认

- **迭代反投影去模糊**：当手写字拍照时手抖或对焦不准导致图片模糊，系统会自动检测并使用迭代反投影算法恢复清晰度。算法通过模拟模糊→计算残差→反向修正的迭代过程，逐步恢复被模糊掉的笔画细节
- **3种去模糊组合策略**：去模糊+锐化（先恢复再强化笔画）、去模糊+CLAHE（先恢复再增强对比度），以及单独去模糊，覆盖不同程度的模糊场景
- **自动触发**：当系统检测到图片模糊度较高时，自动将去模糊策略加入处理流程，无需用户手动操作

### 和上一版的区别
v4.6.1 是稳定性修复（闪退+后台生成），v4.7.0 是识别能力提升 — 新增去模糊算法，让模糊图片的识别率显著提升。之前模糊图片只能靠锐化勉强处理，现在有了专门的去模糊恢复，能还原被模糊掉的笔画结构。

## v4.6.1 (2026-06-23)

### 🔧 稳定性修复 — 闪退问题彻底解决

- **修复字体生成内存溢出**：之前生成7-8个字后会闪退，原因是轮廓缓存无限增长导致内存溢出。现在优化了缓存管理策略，限制缓存上限，生成再多字也不会闪退
- **后台字体生成服务**：字体生成现在独立于屏幕运行，切换页面不会中断生成。首页会显示正在生成的项目和实时进度条
- **增量保存**：每生成一个字就自动保存进度，万一中途退出，下次打开可以继续生成，不用从头来
- **类型错误修复**：修复了 CellResult 类型和 Future 返回类型的编译错误

### 和上一版的区别
v4.6.0 是识别引擎智能化升级（策略组合+边界检测+置信度可视化），v4.6.1 是稳定性加固 — 重点解决字体生成过程中的闪退问题，同时新增后台生成和增量保存，让造字过程更可靠。

## v4.6.0 (2026-06-23)

### 🎯 识别策略组合优化 — 越用越聪明

- **特征签名系统**：将图像特征（对比度/模糊度/笔画粗细）离散化为特征签名（如"C1_B2_T1"），追踪每种特征下各策略的历史性能
- **智能策略排序**：识别时按历史成功率降序排列策略，优先执行高成功率策略，减少无效尝试
- **异步性能更新**：识别成功后指数移动平均更新策略性能，每5次持久化一次，跨会话保留学习成果
- **性能数据持久化**：策略组合性能数据保存到 SharedPreferences，重启后自动加载

### ✂️ 字符边界检测改进 — 粘连断笔通吃

- **水平投影分割**：新增水平方向投影分析，处理上下粘连字符（高宽比>1.8时触发），与垂直投影分割互补
- **断笔合并**：将距离过近的连通域合并（阈值为平均字符高度的25%），迭代合并直到稳定，解决断笔导致的字符拆分问题
- **双重机制**：先分割（垂直+水平投影）再合并（断笔合并），先拆粘连再补断笔

### 🎨 识别结果置信度可视化 — 一目了然

- **精确置信度**：CellResult 新增 preciseConfidence（0.0~1.0），右上角显示百分比（低/中置信度时）
- **候选字面板**：CharacterCell 长按弹出 BottomSheet 候选字面板，显示所有候选字供用户选择
- **候选字指示**：左下角显示候选字数量，有多个候选时可长按查看
- **一键替换**：点击候选字即可替换当前识别结果

### ⚡ 批量识别进度反馈 — 实时掌控

- **修复 ETA 计算**：原 progressValue.toInt() 导致除零，改为基于已完成数量和已用时间的精确计算
- **处理速度显示**：实时显示处理速度（项/秒），让用户清楚知道处理效率
- **精确百分比**：进度百分比精确到小数点后1位
- **自适应布局**：进度信息使用 Wrap 布局，信息多时自动换行

### 📋 识别历史记录 — 有迹可循

- **自动记录**：识别成功后异步写入历史记录（不阻塞主流程），最多保留500条
- **历史查看页面**：搜索栏按字符筛选、统计总览（总次数/平均置信度/纠正率/本地云端分布）、高频字符Top10
- **设置页入口**：设置 → 识别设置 → 识别历史

### 和上一版的区别
v4.5.0 是全链路智能化升级（笔画特征+语言模型+后处理管道+图像增强），v4.6.0 是用户体验和智能调度升级 — 策略选择越用越聪明（特征签名+历史性能排序），字符边界检测更精准（水平分割+断笔合并），识别结果更透明（精确置信度+候选字面板），批量处理更可控（实时ETA+速度），历史记录有迹可循。

## v4.5.0 (2026-06-23)

### 🧠 笔画特征提取增强 — 更懂汉字结构

- **8方向笔画分析**：在原有横/竖/撇/捺4方向基础上，新增横撇/横捺/竖撇/竖捺4个复合方向，方向识别更精细
- **笔画曲率分析**：通过骨架像素的局部方向变化，将笔画分为直线/弯钩/转折/圆弧四类，辅助区分形近字
- **笔画复杂度评分**：综合笔画数量、交叉点密度、前景占比、曲线比例，给出0~1的复杂度分数
- **5维度相似度匹配**：方向(25%)+数量(30%)+结构(20%)+曲率(15%)+复杂度(10%)，形近字区分更准确

### 📖 上下文语言模型 — 前后文推理

- **bigram模型**：从500+常见词库自动构建相邻字对概率，预测"这个字后面最可能是什么"
- **trigram模型**：三字组概率，权重更高，捕捉更长的语境依赖
- **候选评分集成**：6维评分公式新增上下文维度（权重16%），在票数和置信度相近时优先选择符合语境的候选
- **n-gram上下文纠错**：后处理管道新增第3步，利用语言模型自动纠正不符合上下文的识别结果

### ⚡ 后处理管道优化 — 更快更准

- **按代价排序**：错误模式纠正(查表O(1)) → 字典后处理(形近字匹配) → n-gram纠错(语言模型) → 笔画特征辅助(骨架化分析)
- **早退出机制**：错误模式纠正成功后跳过昂贵的笔画分析，减少平均延迟
- **条件执行**：高置信度+已纠正时跳过后续步骤，避免不必要的计算
- **12维置信度校准**：新增n-gram上下文得分(+0.04/-0.03)、错误模式感知(-0.04)、投票一致性(+0.03)

### 🖼️ 图像增强预处理 — 更多策略

- **自适应伽马校正**：根据图片平均亮度自动计算gamma值，过暗提亮、过亮压暗，解决光照不均问题
- **多尺度边缘增强**：3x3+5x5双尺度Sobel算子，同时增强粗笔画结构和细笔画细节
- **笔画感知去噪**：各向异性扩散思想，梯度大处保留原图（笔画边缘），梯度小处用中值滤波（背景噪声）
- **组合策略**：伽马+CLAHE（先亮度归一化再对比度增强）、边缘增强+锐化（多尺度边缘+USM锐化）
- **策略总数**：从25+种扩展到30+种，智能过滤只跑相关的策略

### 和上一版的区别
v4.4.0 是识别精度深度优化（综合评分+质量评估+三层缓存+错误学习），v4.5.0 是全链路智能化升级 — 从笔画特征（8方向+曲率+复杂度）到语言模型（n-gram上下文推理）到后处理管道（按代价排序+早退出）到图像增强（伽马校正+多尺度边缘+笔画感知去噪），每个环节都在向99%识别率逼近。

## v4.4.0 (2026-06-23)

### 🎯 多候选综合评分 — 不再只看票数

- **5 维评分函数**：当 TopN 投票产生多个候选时，不再简单按票数排序，而是综合考虑 5 个维度：票数（35%）、置信度（20%）、字频（15%）、策略多样性（15%）、多尺度一致性（15%）
- **策略多样性加成**：如果一个候选被 3 种以上不同预处理策略识别到，说明它更可能是正确结果
- **多尺度一致性**：多个放大尺寸都识别到同一个字，可信度更高
- **常见字优先**：在票数和置信度相近时，优先选择更常见的汉字

### 📊 图像质量评估增强 — 更懂你的图片

- **亮度分布检测**：通过直方图偏度分析，自动判断图片是过曝还是欠曝
- **背景均匀性检测**：将图片分为 4×4=16 个区块，检测光照不均、阴影、折痕
- **笔画连续性检测**：统计孤立像素比例，评估断笔和飞白严重程度
- **智能增强策略**：背景不均匀时自动应用 Retinex 背景归一化，笔画断裂时自动应用闭运算修复

### ⚡ 批量识别优化 — 批量处理更快

- **特征预分析**：批量处理前提前分析所有图片的特征（尺寸、质量级别）
- **智能分组**：按质量级别 + 尺寸范围分组，同组图片共享预处理经验
- **减少重复计算**：相似特征的图片不再各自独立计算预处理参数

### 💾 三层缓存架构 — 缓存命中率大幅提升

- **精确缓存**：字节哈希 → 结果（已有，保持不变）
- **感知哈希缓存**：缩放到 8×8 灰度计算 pHash，汉明距离 < 10 视为相似图片，实现模糊匹配。同一张图稍微缩放、旋转都能命中缓存
- **持久化缓存**：高置信度（≥80%）的识别结果保存到本地，下次打开 App 自动加载，7 天自动过期

### 🧠 错误模式学习 — 越用越准

- **自动记录**：用户每次纠正识别结果时，系统自动记录"错误→正确"的映射
- **高频纠正**：同一个错误模式出现 2 次以上，后续识别自动应用纠正
- **持久化存储**：错误模式保存到本地，最多 500 组映射
- **集成到后处理管道**：字典纠正 → 笔画辅助 → 错误模式纠正，三重保障

### 和上一版的区别
v4.3.0 是全链路优化（形态学+粘连分割+异常检测+策略权重），v4.4.0 是识别精度深度优化 — 从候选排序（综合评分）到图像评估（3 个新指标）到缓存体系（感知哈希+持久化）到错误学习（自动纠正），每个环节都在向 99% 识别率逼近。

## v4.3.0 (2026-06-23)

### ✂️ 粘连字符自动分割 — 再也不怕写字连笔

- **垂直投影分割**：当检测到连通域宽高比 > 1.8（可能是两个字粘在一起），自动用垂直投影法找到最佳切分点，将粘连字符拆开
- **智能判断**：只有宽度足够（>30px）且确实是横向粘连时才触发，避免误切单个宽字符（如"一"）

### 🔧 更多形态学预处理 — 细笔画、断笔都能救

- **断笔修复**（闭运算）：先膨胀后腐蚀，填充笔画间的小间隙，让断开的笔画重新连接
- **细笔画增强**（自适应膨胀）：自动检测笔画粗细，太细的笔画膨胀加粗，提升识别率
- **开运算去噪**：先腐蚀后膨胀，去除小噪点同时保留主要笔画结构
- **多尺度形态学**：小膨胀 + 闭运算组合，兼顾断笔修复和笔画增强

### 📖 3-4 字短语纠错 — 不只是双字词了

- **短语级上下文**：同音字纠错从只看前后 1 个字，扩展到看前后 2 个字，支持 3 字短语和 4 字成语匹配
- **500+ 短语词库**：新增大量 3 字短语（有没有、是不是、为什么、差不多）和 4 字成语（千变万化、自言自语、实事求是）
- **6 种匹配模式**：2 字 × 2 种 + 3 字 × 3 种 + 4 字 × 2 种，覆盖各种上下文组合

### 🧠 策略权重学习 — 越用越聪明

- **权重持久化**：每个预处理策略的历史成功率保存到本地，下次打开 App 自动加载
- **时间衰减**：每天衰减 1%，防止几个月前的旧数据过度影响当前识别
- **指数移动平均**：策略成功 +0.2，失败 ×0.9，平滑更新不会剧烈波动

### 🛡️ 异常检测 — 自动过滤垃圾结果

- **全面过滤**：标点符号（中英文）、数字、拉丁字母、日文假名、韩文、偏旁部首、全角符号 — 全部自动过滤
- **双重保障**：ML Kit 识别结果层面 + 最终返回层面各做一次异常检测
- **云端也管**：云端 API 返回的结果同样经过异常检测过滤
- **收紧规则**：不再接受 ASCII 字符（手写汉字场景下数字和字母都是误识别）

### 和上一版的区别
v4.2.0 是识别结果后处理的升级（同音字纠错），v4.3.0 是全链路优化 — 从图像预处理（形态学）到字符分割（粘连切分）到结果过滤（异常检测）到策略学习（权重持久化），每个环节都有提升。

## v4.2.0 (2026-06-23)

- **同音字上下文纠错**：批量识别完成后，系统会自动检查每个字和它前后字的搭配关系。比如识别出了"相"但前后字无法组成词时，如果同音字"想"能和后一个字组成"想念"，就会自动替换成"想"
- **300+ 组同音字映射**：覆盖了所有常见拼音组合（如 zhi/chi/shi、de/di/di、zai/zai、zuo/zuo、xiang/xiang 等），确保常见同音字混淆场景都能覆盖
- **500+ 常用双字词库**：内置了工作、成功、已经、因为、可以、没有、开始 等高频双字词，用于判断替换是否合理
- **智能触发**：只在识别置信度中等（30%-85%）时才触发同音字纠错，高置信度的结果直接信任，避免画蛇添足
- **双保险**：与已有的"形近字纠正"互补 — 形近字处理"写错的字"（如"己"→"已"），同音字处理"读对但选错的字"（如"在"→"再"）
- **形近字库扩展 200+ 组**：新增大量手写体高频混淆字对，如清/请/情/晴/睛、折/拆、喝/渴、推/堆 等

### 🎯 识别率优化 — 投票更智能，置信度更准确

- **自适应投票阈值**：根据图片大小和策略数量动态调整提前终止阈值，小图更快返回，大图更充分投票
- **增强置信度校准**：新增 3 个校准因子 — 共识强度（票数占比越高越可信）、TFLite 一致性（双引擎一致加分）、字典验证（常用字加分）
- **字典数据清理**：修复同音字表 4 个重复键、常用词库 100 个重复元素，数据更干净

### 🔧 构建修复
- 修复 tflite_flutter 包 JVM target 兼容性问题（强制子项目使用 JVM 17）

### 和上一版的区别
v4.1.0 是识别引擎层面的升级（双引擎投票），v4.2.0 是识别结果后处理的升级 — 在识别结果出来后，利用上下文语境关系进一步纠错。这是从"认字准"到"用字对"的提升。

## v4.1.0 (2026-06-23)

### 🧠 双引擎识别 — 两个 AI 同时帮你认字

- **TFLite 本地识别模型**：在手机里内置了一个轻量级手写识别 AI，不需要联网就能用。它和 ML Kit 是两套完全不同的识别算法，相当于请了两个专家同时看你的字

- **可自由开关**：设置里新增了 TFLite 模型开关，不想用可以关掉，不影响 ML Kit 正常工作
- **智能降级**：如果模型加载失败（比如手机内存不足），自动退回 ML Kit 单引擎模式，不影响使用

### 和上一版的区别
v4.0.1 重点是白边裁剪+笔画粗细归一化（预处理优化），v4.1.0 是识别引擎层面的升级 — 从单引擎变双引擎，识别结果更可靠。这是架构层面的改进，后续版本会在此基础上继续优化。
